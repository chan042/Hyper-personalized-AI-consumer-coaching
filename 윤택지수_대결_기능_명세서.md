# 윤택지수 대결(Battle) 기능 명세서

> 새로운 에이전트와 원활한 대화를 위한 기준 문서입니다.  
> 프로젝트: `duduk-project` / 프론트엔드: Next.js App Router

---

## 1. 기능 개요

**윤택지수 대결**은 두 사용자가 1:1로 윤택지수 항목을 정해 경쟁하는 기능입니다.

### 핵심 규칙
- 대결은 **매월 1일~15일** 사이에만 시작할 수 있음
- 선택한 항목에 따라 **3개의 미션**이 주어짐
- 미션 1개 성공 시 **+3점** 보너스 (총 최대 +9점)
- 최종 점수 = **선택 항목 기본 점수 + 미션 보너스 점수**
- 대결 최종 우승 시 **500포인트** 지급

## 4. 페이지별 상세 UI 명세

### 4-1. `/challenge-battle/search` — 대결 신청 페이지

**5단계 뷰 구조** (`viewMode` state로 전환):

#### `intro` 뷰 (인트로)
- 문구: `친구와 함께\n윤택지수 대결을 하고\n포인트를 얻어봐요.`
- 버튼 2개 (하단 플로팅):
  - `괜찮아요` → `selection` 뷰로 이동
  - `설명보기` → `guide` 뷰로 이동

#### `guide` 뷰 (캐러셀 가이드, 5슬라이드)
- 상단에 세그먼트 진행 바 (수동 조작, 자동 전환 없음)
- 각 슬라이드: 제목, 설명, 캐릭터 이미지
- 슬라이드 내용 순서:
  1. 진행 방식
  2. 시작 가능 기간
  3. 대결 규칙
  4. 대결 기준
  5. 점수 반영 안내
- 마지막 슬라이드에서 `완료` 버튼 → `selection` 뷰로 이동

#### `selection` 뷰 (상대 검색 + 항목 선택)
- 상단에 `나의 배틀 ID: {battleCode}` 표시
- 상단 문구: `대결 상대를 검색해 선택해주세요.`
- **상대 검색창** → 결과 목록 (이름 + 배틀 ID 표시)
- 사용자 선택 전까지 **항목 선택 영역 + 대결 신청 버튼은 블러 처리**
- 사용자 선택 해제 시 다시 블러 처리
- 항목 선택 문구: `대결 항목 선택해주세요.`
- **항목 목록**: 흰색 카드 버튼 4개 세로 나열 (토스 스타일)
  - 동일 항목 재클릭 시 선택 해제됨
- **대결 신청 버튼**: 플로팅 고정 (하단 바 위, 인트로 페이지 설명보기 버튼과 동일 위치)
  - 상대 + 항목 미선택 시: 회색
  - 모두 선택 시: 민트색(primary)으로 활성화

#### `request_pending` 뷰 (내가 신청 후 대기)
- 문구: `{상대이름}님에게 대결을 신청했어요.`
- 보조 문구: `수락을 기다리는 중입니다.`
- 버튼:
  - `신청 취소`
  - `확인`

#### `request_received` 뷰 (상대가 나에게 신청)
- 문구: `{상대이름}님이 대결을 신청했어요.`
- 선택한 대결 항목 표시
- 버튼:
  - `거절`
  - `수락`

### 4-2. `/challenge-battle/progress` — 대결 진행 중 페이지

**UI 구성:**
- 상단: `대결 종료까지 D-{숫자}` 표시
- 중앙: 두 사용자 프로필 아바타 + VS 배지 + 현재 점수
- 미션 목록 (진행 중 / 완료로 구분):
  - 클릭 시 `MissionDetailModal` 팝업 오픈
  - 배경 스크롤 차단 (스크롤바 width 보상으로 레이아웃 밀림 방지)

**미션 status 값**: `'진행중'` | `'완료됨'`

**컴포넌트**: `MissionDetailModal`
- 경로: `frontend/src/components/challenge-battle/MissionDetailModal.js`
- `isOpen`, `onClose`, `mission` props 받음
- 팝업 열릴 때 `document.body.style.overflow = 'hidden'` + paddingRight 보상으로 배경 고정

### 4-3. `/challenge-battle/result2` — 결과 페이지

**사용 시점**: `대안 행동 실현도`, `성장`, `건강 점수`, `챌린지` 항목 선택 시

**UI 구성:**
1. 승자/패자 프로필 + 결과 문구
   - 일반 승리: `{winner}님 Win!`
   - 무승부: `무승부`
2. **선택 항목 카드** (카드 제목 = 선택한 항목명, 예: `건강 점수`):
   - 점수 바 (최대 배점 대비 %)  
   - **미션 결과** 소제목 (카드 제목과 동일 폰트 크기/굵기)
   - 미션 3개 카드 목록 (흰색 카드, 테두리, 이름 + 성공자 이름 또는 `동시 달성(무승부)`)
     - 성공자 색상: 내가 이기면 민트, 상대가 이기면 오렌지
     - ~~+3 알약 뱃지 없음~~ (제거됨)
   - 총점 = 항목 기본 점수 + 미션 보너스
3. **전체 점수 결과 카드** (`점수 결과` 제목):
   - 7개 항목 전부 표시
   - 해당 선택 항목 점수는 실제 대결 점수 반영
4. `확인완료` 버튼

## 6. 디자인 원칙 & 스타일 가이드

### 색상
| 토큰 | 용도 |
|---|---|
| `var(--primary)` | 민트색 (두둑 브랜드 컬러), 내 점수 바, 승리자 강조 |
| `var(--primary-light)` | 연한 민트 (사용 지양 — 점수바에서 제거됨) |
| `#cbd5e1` | 상대방 점수 바 |
| `#d97706`, `#f59e0b` | 상대방 (오렌지) 강조 |

### 점수 바 스타일
- **result2 page (선택 항목)**: `height: 10px`, `borderRadius: 6px` (더 크게)
- **result2 page (전체 점수)**: `height: 6px`, `borderRadius: 4px`
- 항상 **최대 배점 대비 비율(%)**로 width 계산

### 버튼 스타일 패턴
- `대결 신청` 버튼: 플로팅 고정, 하단 바 위에 위치, 항상 전체 너비
- 상태: `background: #e2e8f0` (비활성) → `var(--primary)` (활성)

## 10. 알아두어야 할 기술적 이슈 & 해결 이력

### 미션 팝업 배경 밀림 현상
- **문제**: `document.body.style.overflow = 'hidden'` 적용 시 스크롤바가 사라지며 레이아웃이 우측으로 밀림
- **해결**: `const scrollBarWidth = window.innerWidth - document.documentElement.clientWidth` 계산 후 `document.body.style.paddingRight = scrollBarWidth + 'px'` 보상 적용
- **파일**: `MissionDetailModal.js`

### GlobalHeader 연동
- `search`, `progress`, `result2` 3개 배틀 페이지 모두 자체 헤더 없이 `GlobalHeader`에서 경로 감지해 처리
- `GlobalHeader.js`의 `getHeaderContent()` 함수에 각 `pathname`별 케이스 추가

### 대결 신청 버튼 플로팅
- `position: fixed`, `bottom: calc(...)`, 좌우 0으로 고정
- 하단바(`BottomNavigation`) 위 여백 확보: `bottom: calc(80px + env(safe-area-inset-bottom, 0px))`
- 버튼 너비: `maxWidth: '340px'`, margin auto로 중앙 정렬
